# Wortklar

Минималистичный русско-немецкий и немецко-русский переводчик для изучения языка.

## Идея

Wortklar не только переводит, но и объясняет структуру языка:

- показывает артикль и род существительных;
- показывает множественное число;
- выводит три основные формы глагола;
- отмечает вспомогательный глагол и управление;
- объясняет различия употребления;
- даёт живые примеры с переводом;
- позволяет сохранять слова для повторения.

## Текущий статус

Рабочий переводчик с интерфейсом настроек, Groq API и структурированным
грамматическим разбором. API-ключ хранится только как зашифрованный секрет
`apikey` в Cloudflare и доступен серверной Pages Function через `context.env`.

По умолчанию используется бюджетная production-модель `openai/gpt-oss-20b` со
строгим JSON Schema. Для более сложных фраз в настройках доступна
`openai/gpt-oss-120b`.

## Запуск

1. Установить Node.js 20 или новее.
2. Выполнить `npm install`.
3. Выполнить `npm run dev`.
4. Открыть `http://localhost:3000`.

## Развёртывание на Cloudflare

Интерфейс экспортируется в статическую папку `out`, а серверный маршрут
`/api/translate` работает как Cloudflare Pages Function из каталога `functions`.
Так сайт и API развёртываются вместе в существующий Pages-проект.

Для Git-интеграции Cloudflare Pages:

- production branch: `main`;
- root directory: `/`;
- build command: `npm run build`;
- output directory: `out`.

Локальная проверка Pages-сборки:

```bash
npm ci
npm run preview
```

Ручное развёртывание из авторизованной среды:

```bash
npm ci
npm run deploy
```

Папка публикации также закреплена в `wrangler.jsonc`, поэтому новый Git-деплой
может получить правильную конфигурацию непосредственно из репозитория. Готовая
папка `out` хранится в репозитории, поскольку текущий Pages-проект не запускает
build command перед публикацией.
